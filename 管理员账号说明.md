# 管理员账号创建说明

## 自动管理员账号创建机制

系统已实现自动管理员账号创建功能，通过特定手机号前缀识别：

### 管理员账号规则
- **手机号前缀**：`188` 开头的手机号
- **自动权限**：注册时自动获得管理员权限
- **用户名**：显示为 `管理员XXXX`（XXXX为手机号后4位）
- **特殊标识**：
  - `role: 'admin'`
  - `isAdmin: true`
  - 头像使用管理员专用头像
  - 余额设置为 999999
  - 评分固定为 5.0

### 使用方法

#### 注册管理员账号
1. 在注册页面输入以 `188` 开头的手机号
2. 设置密码（6-20位）
3. 完成注册
4. 系统会自动识别并授予管理员权限

#### 示例管理员账号
- 手机号：`18812345678`
- 密码：任意6-20位密码
- 注册后自动成为管理员

#### 登录管理员账号
- 使用已注册的管理员账号直接登录
- 系统会自动识别管理员身份

### 普通用户账号
- 非 `188` 开头的手机号注册为普通用户
- `role: 'user'`
- `isAdmin: false`

### 现有测试账号
- 手机号：`13800138000`
- 密码：`123456`
- 权限：普通用户

## 技术实现

### 修改的文件
- `src/stores/auth.js`

### 核心逻辑
1. **注册时检查**：`mockRegisterApi` 函数检查手机号前缀
2. **登录时检查**：`mockLoginApi` 函数检查手机号前缀
3. **权限标识**：添加 `role` 和 `isAdmin` 字段
4. **持久化存储**：权限信息保存到 localStorage

### 权限字段说明
```javascript
{
  role: 'admin' | 'user',     // 用户角色
  isAdmin: true | false        // 是否为管理员
}
```

## 注意事项
1. 管理员权限目前主要用于前端显示区分
2. 如需后端权限验证，需要在 API 层面添加相应逻辑
3. 可以根据需要修改管理员手机号前缀规则
